SaaS-платформа парсинга и мониторинга

Описание проекта

Разработка и поддержка SaaS-платформы для парсинга и мониторинга данных из 80+ внешних источников (маркетплейсы, сайты поставщиков). Платформа используется несколькими внутренними командами (аналитика, BI, отдел парсинга) как единая точка управления и наблюдения за сбором данных.

Моя роль
  • Backend-архитектура сервиса
  • Разработка и поддержка ключевых микросервисов
  • Стабильная работа системы в production
  • Работа с production и preprod: деплои, мониторинг, hotfix/rollback
  • Code review и помощь младшим разработчикам
Микросервисная архитектура
  • Core SaaS Backend (Django + DRF) - управление пользователями, источниками, парсерами, правами доступа
  • Parser Orchestrator (FastAPI) - планирование и запуск парсеров, работа с очередями
  • Parser Workers - асинхронные воркеры для сбора данных
  • Monitoring Service - отслеживание статусов, ошибок, рассылка уведомлений
  • Data Processing Service - валидация, нормализация и агрегация данных
  • Integration Layer - обмен событиями с другими системами через Kafka
Технологии

Django FastAPI DRF PostgreSQL ClickHouse Kafka RabbitMQ Celery CI/CD

Результаты
  • Обработка данных из 80+ источников
  • Стабильность сбора ~95%
  • Платформа используется 3 командами

Система мониторинга и тестирования

Описание проекта

Разработка сервиса совместно с другим разработчиком для автоматического тестирования парсеров и мониторинга их работы. Система отправляет уведомления через Telegram и push-уведомления, а также дублирует их в Django интерфейс для просмотра падений и конкретных ошибок.

Проблема

Раньше при падении парсера приходили только уведомления с указанием сервера, но без детальной информации об ошибке. Время на починку парсера могло занимать до недели.

Решение
  • Разработала новый процесс: написание unit и интеграционных тестов для каждого парсера
  • Интеграция тестов с платформой мониторинга
  • Автоматическое выявление проблемных мест в коде
  • Детальные логи и stack traces в уведомлениях
  • Dashboard для визуализации ошибок
Технологии

Python Django pytest Telegram Bot API CI/CD Monitoring

Результаты
  • Сокращение MTTR: время починки парсера снизилось с недели до 1-2 дней
  • Автоматическое выявление проблем на этапе тестирования
  • Детальная информация об ошибках сразу при падении

API интеграции отделов

Описание проекта

Проектирование и разработка с нуля API на FastAPI для обмена данными между отделами университета. До этого проекта отделам приходилось физически ходить друг к другу для запроса документов, что было крайне неэффективно.

Проблема

Отделы не имели налаженного цифрового взаимодействия. Для получения данных о студентах или документов из другого отдела сотрудникам приходилось лично идти в этот отдел и подтверждать свою личность и права доступа.

Решение
  • Спроектировала единое API для взаимодействия отделов
  • Система аутентификации и авторизации
  • Управление правами доступа к различным типам данных
  • Логирование всех запросов для безопасности
  • Руководство командой из 2 разработчиков
  • Документация API
Технологии

FastAPI PostgreSQL REST API OAuth Documentation Architecture

Моя роль
  • Проектирование архитектуры системы
  • Руководство командой разработки
  • Делегирование задач разработчикам
  • Разработка ключевых компонентов
  • Code review
Результаты
  • Цифровизация взаимодействия между отделами
  • Автоматизация запросов данных
  • Контроль доступа и безопасность
  • Экономия времени сотрудников
